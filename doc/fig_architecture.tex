%% ── Caption ─────────────────────────────────────────────────────────
\caption{%
  System architecture of the proposed \textsc{Graph-Matcher-PPO}
  framework.
  At each decision epoch $t$, the GNN-based PPO policy processes the
  joint observation $\mathcal{O}(t)$---comprising the current task DAG
  $\mathcal{G}_T$, the dynamic resource graph $\mathcal{G}_R$, and the
  six-dimensional scheduling feedback $\mathbf{f}(t)$---and selects a
  topology-shaping action $a_t$.
  The action either decomposes a task via \textsc{Split}$(v_i)$,
  collapses two tasks via \textsc{Merge}$(v_i, v_j)$, or leaves the
  graph unchanged (\texttt{noop}), yielding a modified DAG
  $\mathcal{G}_T'$.
  The deterministic greedy scheduler then maps $\mathcal{G}_T'$ to a
  concrete UAV assignment using the instantaneous bandwidth matrix
  $\mathbf{B}(t)$ and effective CPU vector $\tilde\phi_k(t)$ supplied
  by the UAV swarm dynamics; the resulting workload
  $(\rho_k(t), \tau_k(t))$ drives battery depletion
  $\beta_k(t{+}1)$.
  The scheduling outcome is encoded into the updated feedback vector
  $\mathbf{f}(t{+}1)$, closing the adaptive loop.%
}
